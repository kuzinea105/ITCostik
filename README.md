# ITCostik

Корпоративный локальный ассистент для компании ITCost (n8n + Ollama).  
Цели: быстрые ответы сотрудникам, генерация таблиц/Excel, подготовка документов (КП, тендеры, договоры), интеграции с корпоративными чатами.

## Что в репозитории
- `workflows/` — экспорты n8n workflow (без секретов)
- `docs/` — описание логики, форматов ответов и правил
- `scripts/` — утилиты для тестов/отладки (если добавим)

> ⚠️ В репозитории **нет токенов** и **нет .env**. Все ключи и URL задаются через переменные окружения на стенде/у заказчика.

## Архитектура (кратко)
1) Сообщения приходят из корпоративного чата (Webhook)
2) n8n агрегирует сообщения, вложения, выбирает лидера (anti-dup)
3) По типу запроса запускается нужный сценарий:
   - корпоративный ассистент (LLM)
   - OCR / распознавание документов
   - парсер логов (отдельный модуль)
4) Ответ возвращается в чат (текст / файл CSV/TXT/DOCX/XLSX)

## Требования к стенду
- Ubuntu 24.04 (или аналог)
- Docker / docker compose
- n8n (self-hosted)
- Ollama (локальные модели)

## Настройка (в общих чертах)
1) Разверни n8n и Ollama на тестовом стенде или у заказчика
2) Настрой переменные окружения (пример: `.env.example`)
3) Импортируй workflow в n8n
4) Проверь базовый сценарий (сообщение → ответ)

## Примечания по безопасности
- Никогда не коммитьте токены, ключи и приватные URL в workflow JSON.
- Используйте env vars для токенов (например: `YANDEX_BOT_TOKEN_*`, `OCR_URL`, `RAG_API_URL`).
